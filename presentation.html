Arquitectura del Sistema

  • Edge Devices: Nodos que publican datos de sensores vía red inalámbrica.
  • MQTT Broker: Middleware de mensajería ligero.
  • Backend Flask: Suscrito a los tópicos, procesa y almacena datos.
  • Machine Learning: Modelo pre-entrenado clasifica la ocupación en tiempo real.
  • Dashboard: Interfaz web para visualización.

Despliegue con Docker

Implementación mediante contenedores, reemplazando la Raspberry Pi física.

docker-compose.yml 
services:
  mqtt:
    image: eclipse-mosquitto:latest
    container_name: smart-building-mqtt
    ports:
      - "1883:1883"
    volumes:
      - ./mosquitto.conf:/mosquitto/config/mosquitto.conf:ro

  web:
    build: .
    container_name: smart-building-web
    ports:
      - "5001:5000"
    environment:
      - MQTT_BROKER=mqtt
      - DB_PATH=/app/data/building.db
      - OCCUPANCY_MODEL_PATH=/app/occupancy_model.pkl
Dockerfile 
FROM python:3.11-slim

WORKDIR /app

# Install python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application files
COPY . .

# Create data directory for persistent SQLite
RUN mkdir -p /app/data
RUN python init_db.py /app/data/building.db

EXPOSE 5000
CMD ["python", "app.py"]

Desempeño Machine Learning

Evaluación de algoritmos para predecir la ocupación basándose en las variables ambientales de los sensores.

  • Comparativa de precisión entre Random Forest, XGBoost, etc.
  • Alto rendimiento logrado en validación cruzada.

Limitaciones Encontradas

Desafíos en Hardware y Entorno

  • Calibración del sensor MQ135:
    Gran dificultad en la calibración por la enorme variabilidad de las lecturas. Los valores fluctuaron drásticamente al comparar aulas vacías con aulas llenas, y al contrastar el entorno universitario con el doméstico.
  • Etiquetado Manual:
    La dependencia de agregar etiquetas de manera manual para la recolección de datos hizo el proceso propenso a sesgos e imprecisiones.

Posibilidades de Mejora

Autocalibración ML

Sugerimos implementar Machine Learning para un periodo de calibración inicial (algunas horas) y una posterior autocalibración continua y adaptativa.

Fusión Sensorial (PIR/Luz)

Añadir sensores PIR de movimiento, sensores de presencia humana, o de luminosidad para correlacionar los picos de CO2 y aumentar la asertividad.

Sensores Adicionales (MQ8)

Incorporar otros sensores de la familia MQ (como MQ8) para monitorear riesgos y fugas peligrosas en los laboratorios de la universidad.

Validación Multientorno

Entrenar el modelo con datasets más diversos incluyendo múltiples infraestructuras (no solo la casa y el aula local).
